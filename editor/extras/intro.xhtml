XTiger XML Demo Editor

This is an application that can be run directly from your local file system (URL starting with file:///) with Firefox. For Firefox >= 3 you need to authorize access to local URL from the XMLHTTPRequest object. To do this type about:config in the URL bar then click on security.fileuri.strict_origin_policy to set it to false (source: mozillazine.org). You most probably will not be able to use it from the local file system on the other browsers (such as Safari or Internet Explorer) as they will not allow the application to access local files. For instance Safari will just show them in the finder.

On the contrary if you start the application from a Web server (URL starting with http://), including a WebDav server, it should work on most of the browsers. For instance it works for Internet Explorer, but only if you use the version of the application that uses no iframe (see the window title bar).

  • Page : enter a file URL to open a template file as a document to edit (or use Page shortcut)
  • Page shortcut : select directly a predefined template file
  • [Visualize] : open the selected template file
  • [Dump] : dump the XML content of the current document inside a popup window.
  • [Input] : enter XML data from a popup window to the current document
  • [Save] : prompts a dialog to select a local XML file name and saves the XML content to it (Firefox only)
  • [Load] : prompts a dialog to select a local XML file name and loads XML data from it (Firefox only)

The [Preferences] button opens a pannel with several actions

  • Template repository [Ok] : enter the path for the default XTiger XML template folder (displayed in Page shortcut)
  • File name [save] [load] [post]: save/load XML content in the current document, depending on the browser, [save] and [load] may work only if you have opened the application from a local server such as the one proposed in scripts/server. [post] asks the server to only dump the data without saving it.

NOTE: you can use the application directly from a local volume (document URL starting with file://), or you can copy it and launch it from a Web server such as the one in scripts/server.

By default template files listed in the "Page shortcut" popup menu are stored in the "templates" folder.
